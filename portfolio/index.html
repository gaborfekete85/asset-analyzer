Multi-Agent Crypto Research with LangGraph

A supervisor agent orchestrates five specialist sub-agents in parallel — each a self-contained LangGraph node with its own skill prompt — and a synthesizer joins their reports into a single composite-scored markdown deliverable.

Compiled LangGraph topology, rendered via graph.get_graph().draw_mermaid_png().

What this is

asset-analyzer takes a single cryptocurrency ticker — BTC, ETH, SOL, anything CoinGecko knows — and produces a deeply-researched markdown report scored 0–100 across five independent dimensions. Each dimension is owned by a dedicated sub-agent with its own focused skill prompt, search budget, and scoring rubric.

The five specialists are On-Chain Analytics, Tokenomics, Sentiment & Momentum, Technical Analysis, and Fundamental Analysis. They run concurrently, then a synthesis step weighs their findings, flags cross-dimensional convergence and divergence, and writes the final composite report.

Skills as source of truth

Each sub-agent's behaviour comes from a single SKILL.md file loaded verbatim as its system prompt. Tweak the markdown, re-run, behaviour changes — no code edit needed.

Provider-agnostic LLMs

Built on LangChain's init_chat_model so the same graph runs on Anthropic, OpenAI, Google, or local Ollama models — picked per role via env vars.

Parallel by construction

Five sibling edges from discovery to the sub-agents place all five in the same Pregel superstep. They actually run concurrently, not in a loop.

Composable scoring

Each sub-agent emits a 0–100 score with five 0–20 sub-dimensions. The synthesizer combines them into a single composite score with an A+/A/B/C/D/F grade and a Buy/Hold/Avoid signal.

What does LangGraph do here?

LangGraph is a graph-based orchestration framework on top of LangChain. You declare nodes (units of work — usually an LLM call, a tool call, or a small Python function) and edges (how state flows between them). The runtime is a Pregel-style scheduler: in each "superstep" every currently-runnable node executes in parallel, then the scheduler advances.

For this project that gives us three things essentially for free:

  • Parallel fan-out / fan-in. Five edges from discovery to the sub-agents and five edges back into synthesize are all the wiring needed to run the specialists concurrently and join their results.
  • Structured shared state. A typed AnalyzerState dictionary flows through the graph. Each node writes to its own key, so concurrent writes never collide.
  • Subagents as react loops. Each sub-agent is a create_react_agent — an LLM that can call the Tavily search tool until it has enough evidence, then emits the final markdown report. The supervisor doesn't care what happens inside.

The mapping from the original Claude-Code skill design is essentially one-to-one: orchestrator → supervisor graph, sub-agents → subgraph nodes, skill markdown → system prompts.

i
No live demo. This page intentionally doesn't expose a hosted endpoint — a single end-to-end run hits external LLM and search APIs that carry real per-request cost, and a public button would be expensive to keep open. Clone the repo, point it at your own API keys, and run it locally — a full analysis takes 2–4 minutes and finishes for cents on the cheap-OpenAI preset.
